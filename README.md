# prompt

我想做我的大学四年生活纪念照片网站。我的要求如下：首页是一个动态的球，效果要和我给你的这个mp4文件展示的一样，这个球由所有我上传的图片模块组成（每个图片模块由一张照片、拍摄照片、我的感想、tag等等组成），然后屏幕右侧有一个发光的悬浮按钮，点击之后所有照片向四周散开/飞走，同时进入主页面。主页面的布局以及风格参考我给你的第一张图。主标题和辅助文字你自己设计，副标题的部分设计一个返回首页的按钮；此外有四个按钮：大一、大二、大三、大四、随机记忆抽取（五个按钮的大小一样，随机记忆抽取这个按钮在第三行正中）。每个按钮点进去就是对应的这一年，列出所有月份按钮，这里的月份怎么设计和布局你帮我想，风格也可以和主页面的风格一致。同时也要有返回上一级的按钮。每个月份按钮点进去就陈列所有照片模块，这里的照片模块怎么设计布局你帮我想（注意我的照片可能有点多），风格同样可以参考主页面。这里也要有返回上一级的按钮。还要加一个新增照片模块的功能，这样我就可以在网页新增照片，而不用每次都跑到源代码里去添加照片。帮我做一个设计方案出来


# 交互方案

**照片悬停效果**

适用于首页球体里的照片卡片：

- 鼠标经过某张照片时，卡片轻微放大，比如 `scale(1.12)`。
- 卡片周围出现更明显的光圈。
- 光圈可以是柔和的金黄色或白黄色外发光。
- 当前卡片层级提高，避免被其他照片遮挡。
- 球体自动旋转可以暂时减慢，让用户更容易点击。
- 鼠标移开后，卡片恢复原大小，光圈减弱。

效果感觉是：

> 这张记忆被你“点亮”了。

**照片点击聚焦效果**

适用于首页球体和月份照片页：

- 点击任意照片模块后，照片卡片从原位置飞到屏幕正中。
- 卡片放大成详情卡。
- 背景出现半透明遮罩。
- 原页面内容保持在后面，但弱化、模糊或变暗。
- 详情卡展示完整信息：
  - 大图
  - 标题
  - 拍摄日期
  - 拍摄地点
  - 我的感想
  - tags
  - 所属年级 / 月份
- 点击屏幕其他区域后：
  - 详情卡缩回原位置。
  - 遮罩消失。
  - 原照片卡片恢复原状态。

首页球体点击后，可以暂停球体旋转；关闭详情后再继续缓慢旋转。

**月份照片页交互**

进入某个月份后，所有照片模块以瀑布流或网格方式陈列。这里的照片卡片也保持同样逻辑：

- 鼠标经过：轻微上浮、放大、外发光。
- 点击：卡片跳到屏幕正中放大。
- 显示详细信息。
- 点击遮罩区域恢复原位。

这样用户不管在首页球体，还是在照片列表页，操作体验都是一致的，学习成本很低。

**详情卡视觉建议**

详情卡可以延续参考图的海报感：

- 粗黑边框。
- 错位黑色阴影。
- 轻微旋转角度。
- 背景用白色或浅黄色。
- tag 使用彩色小标签。
- 图片占详情卡上半部分，文字信息在下半部分。
- 感想文字区域可以滚动，避免长文字撑爆卡片。

**动效细节**

- 悬停放大：快速，约 `180ms`。
- 光圈增强：约 `200ms`。
- 点击飞到中心：约 `450ms`，带一点弹性。
- 关闭回原位：约 `350ms`。
- 遮罩淡入淡出：约 `200ms`。

推荐用 `Framer Motion` 或 `GSAP Flip` 实现这个“从原位置飞到中心再返回”的动画。尤其是 `GSAP Flip` 很适合做这种卡片位置切换，效果会自然很多。

更新后，这个网站的核心交互可以概括成：

> 悬停时点亮记忆，点击时把这段记忆请到屏幕中央。
